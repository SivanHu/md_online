### 让 Codex 的工作记录更易读，并增加 Markdown 原样上传

#### 我想做什么

我希望每日总结真正记录“我让 Codex 做了什么”，并且以后打开时能够轻松读懂，而不是看到一份充满文件名、接口和函数的技术报告。一天内可能处理多个项目，所以总结还需要按项目区分。

除此之外，我有时已经让 Codex 写好了一篇 Markdown，只想把成品直接上传到在线文档，不希望它再被套用每日总结格式。

#### Codex 帮我做了什么

Codex 重新设计了每日总结 skill，把内容改成“我想做什么、Codex 帮我做了什么、最终结果、对我有用的知识”。同步时会识别当前项目，把当天的工作写入该项目自己的日报文件；同一个任务再次记录时会更新原内容，而不是不断生成重复条目。

Codex 还新增了一个独立的 Markdown 上传 skill。它只负责把已经写好的文件原样上传，不总结、不改写，也不会擅自覆盖线上已有文档。

#### 最终结果

- 每日总结现在更像可回顾的个人工作日志，而不是 Git 变更报告。
- 不同项目拥有各自的日报目录，不再共享同一篇文档。
- 可以使用 `$update-daily-summary` 记录 Codex 完成的工作。
- 可以使用 `$upload-markdown` 原样上传已经生成的 Markdown 文件。

#### 对我有用的知识

“记录工作过程”和“上传文档成品”是两种不同需求，拆成两个 skill 后，调用目的更明确，也能避免 Codex 对已有文档进行不必要的改写。项目级 skill 放在 `.agents/skills` 中，可以跟随仓库在 Windows、VMware 和 VS Code Codex 之间同步。
